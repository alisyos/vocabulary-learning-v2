콘텐츠 생성 페이지에 기능을 추가 하고자 합니다.
구체적으로 문단별 문제 생성 단계를 추가하고자 합니다. 순서는 ‘어휘 문제 검토’ 이후에 추가 됩니다. 즉 단계의 순서는 아래와 같습니다.

###
1단계 지문 생성 > 2단계 지문 검토 > 3단계 어휘 문제 생성 > 4단계 어휘 문제 검토 > 5단계 문단 문제 생성 > 6단계 문단 문제 검토 > 7단계 종합 문제 생성 > 8단계 종합 문제 검토 > 9단계 저장

###
[5단계 문단 문제 생성 단계 정의]
* 2단계에서 검토 완료된 문단 중 생성할 문단을 선택하는 페이지. 디폴트는 모두 선택되어 있음
* 생성요청 시 문제 형태를 선택할 수 있으며, 형태는 아래 5가지와 ‘Random’ 중 하나를선택 할 수 있으며, 디폴트는 랜덤 임.
* 기본적으로 5개의 문제가 생성되며, 특정 형태를 지정한 경우는 해당 형태의 5개 문제가 생성되고, ‘Random’을 선택한 경우는 5가지 형태의 문제가 1개씩 생성됨
* 문제 형태 정의
1) 어절 순서 맞추기 (객관식)
•  형식: 문장의 어절들이 순서 없이 제시됨 → 올바른 순서로 배열
•  목적: 어휘 배열 능력과 문장 구성 이해도 향상
•  예시:
•	주어진 어절: 놀았다 / 우리는 / 공원에서
•	정답: 우리는 공원에서 놀았다
2) 빈칸 채우기 (객관식)
•  형식: 주요 어휘가 빠진 문장 제공 → 선택지에서 빈칸에 적절한 단어 선택
•  목적: 어휘의 의미와 문맥에 따른 적절성 파악
•  예시:
•	문장: 그는 매우 ( ) 얼굴로 나를 쳐다보았다.
•	보기: ① 기쁜 ② 날카로운 ③ 반가운 ④ 어두운
•	정답: ④ 어두운
3) 유의어 고르기 (객관식)
•  형식: 문단에서 특정 문장과 단어를 제시하고, 유사한 의미의 단어 선택
•  목적: 어휘 확장 및 의미군 이해
•  예시:
•	단어: 그녀는 [커다란] 공을 가지고 있었다.
•	보기: ① 작은 ② 넓은 ③ 거대한 ④ 얇은
•	정답: ③ 거대한
4) 반의어 고르기 (객관식)
•  형식: 문단에서 특정 문장과 단어를 제시하고, 단어의 반대 의미 단어 선택
•  목적: 어휘 관계에 대한 이해 강화
•  예시: 
•	단어: 예상되는 그의 미래는 [밝다]
•	보기: ① 빛나다 ② 환하다 ③ 어둡다 ④ 깨끗하다
•	정답: ③ 어둡다
5) 문단 요약 (객관식
•  형식: 문단의 주요 내용을 대표하는 문장 선택
•  목적: 어휘의 의미 확장 및 핵심 정보 추출 능력 강화
•  예시:
•	보기 중에서 문단을 가장 잘 요약한 문장 고르기

* 문제 생성 프롬프트 및 출력값 등은 기존 어휘 문제 및 종합 문제 생성 형태를 참고하여 유사하게 적용

###
[6단계 문단 문제 검토 단계 정의]
* 생성된 문단 문제를 확인하고, 필요 시 수정할 수 있는 단계. ‘어휘 문제 검토’, ‘종합 문제 검토’ 페이지’와 유사하게 생성할 것
